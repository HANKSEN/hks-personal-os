# 两条用户旅程

Personal OS 用同一个 GitHub 链接服务不同经验水平，但不会要求用户先学习同一套术语。

## 旅程 A：从零使用 Agent 的小白

### 原来的问题

- 不知道 AI 工作目录建在哪里；
- 不知道资料、过程稿和完成产物怎么分；
- 每次都要先想“我应该进入哪个文件夹”；
- 害怕 Agent 直接改乱文件。

### 从安装到第一次使用

1. 把 GitHub 链接和安装请求交给 Agent；
2. 只确认 Skill 安装位置，默认不安装 CLI；
3. 回答“新建一套”；
4. 确认一个缺失或空的绝对路径；
5. 可选填写 1–3 个长期领域，也可以跳过；
6. 系统自动创建目录、`START_HERE.md` 并运行 Doctor；
7. 用户给一篇文章、一个困惑或一项真实任务；
8. AI 最多追问一个会改变方向的关键问题，在隔离区工作并展示第一次正式变更。

### 用户真正需要学会的事

- “不知道放哪里”不是用户的失败，直接表达意图或交给 Inbox；
- Project 有结束条件，Area 是长期责任，但 AI 可以先提议；
- 正式结果会按 Knowledge、Experience、Principles、Artifacts、Data 沉淀；
- 看懂路径和变化范围后再确认，不需要理解命令参数。

### 价值

把第一次 Agent 使用从“自己建目录、学方法、记命令”变成“表达真实目标—确认路径—完成一个任务”。系统同时建立以后可持续的归档习惯。

## 旅程 B：已有 Agent 经验，但文件已经混乱

### 原来的问题

- 不同 Agent 自行创建 `output`、`draft`、`final-v2`、`research` 等目录；
- 同一作品有多份副本，外部资料和自己的理解混在一起；
- 不知道哪些还在进行、哪些可以归档；
- 想整理，又担心 Agent 批量误删或移动资产。

### 从安装到治理

1. 安装同一个 Skill；
2. 回答“整理已有目录”；
3. 指定且只指定一个旧目录；
4. 完成独立备份并授权只读审计；
5. 确认一个独立的新 Personal OS 目标；
6. 获得现状报告、路径映射、未决项和归档指引；
7. 可以选择 audit-only，完全不复制正式资产；
8. 如果继续，只审批看过的映射，分批复制到新目标；
9. 每批经过哈希、Changeset、冲突、受保护上下文和 Doctor 验证；
10. 保留旧目录，直到人工抽检和备份恢复都通过。

### 价值

- 先获得“发生了什么”的事实清单，再讨论“应该放哪里”；
- 形成 PARA 物理归属和五类资产意义的统一规则；
- 不把 AI 的文件名猜测伪装成用户确认；
- 旧目录不变，新目录可逐批验收，显著降低一次性重构风险；
- 最终交付的不只是整理结果，还有一份今后 Agent 都能遵循的归档指引。

## 同一个日常入口

两类用户初始化完成后，都可以只用三句话：

```text
把这份内容当成新输入，帮我判断应该怎么处理。
继续推进我正在做的这件事，只读取必要上下文。
复盘这次行动，把事实、经验和可复用方法分开。
```

使用水平影响 Agent 解释的深度，不影响底层安全边界和资产规范。
