# 从这里开始

Personal OS 帮你解决两件事：不知道应该把资料和 AI 产物放在哪里，以及已有文件越来越乱、难以继续使用。

你不需要先学习目录规则。日常只需要表达真实目标，Personal OS Skill 会先澄清用途，再读取必要上下文，并在安全工作区内准备结果。

## 最小使用闭环

1. 把一个文件放进 `00_Inbox/`，或者直接告诉 Agent 你想做什么。
2. Agent 判断它属于正在完成的 Project、长期负责的 Area，还是外部 Resources。
3. AI 先在 `99_AI/hosts/<host-id>/runs/` 内工作，不同 Agent 的临时内容互不混放。
4. 需要长期保存时，Agent 展示准备写入的位置和变更内容。
5. 你确认后，内容才进入正式空间。

## 四个物理空间

| 空间 | 什么时候使用 |
|---|---|
| `10_Projects/` | 有明确结果和完成条件的工作 |
| `20_Areas/` | 需要长期经营的责任和资产 |
| `30_Resources/` | 已分类但尚未吸收、也没有当前负责人的外部资料 |
| `90_Archive/` | 已结束或暂时不再活跃的内容 |

不知道放哪里时，不要停下来分类：把它交给 Agent 或放进 `00_Inbox/`。

## Area 内的五类资产

| 资产 | 回答的问题 |
|---|---|
| Knowledge | 我理解了什么？ |
| Experience | 我在什么时间和场景下做了什么，结果怎样？ |
| Principles | 哪些规则、方法或 SOP 值得再次使用？ |
| Artifacts | 我完成、发布或交付了什么？ |
| Data | 哪些结构化事实和指标支持判断？ |

你不需要在任务开始时选择这些类型。先说清目标，Agent 会提出归档建议。

## 可以直接对 Agent 说

- “把这份内容当成一个新输入，帮我判断应该怎么处理。”
- “继续推进我正在做的这件事，只读取必要的上下文。”
- “复盘这次行动，把事实、经验和可复用方法分开。”

## 安全提示

- 首次让 Agent 访问有价值的既有文件前，先做独立完整备份，并实际验证可以恢复。
- AI 默认先在 `99_AI/` 内起草；正式变更需要检查目标路径和 Changeset。
- Personal OS 的 Undo、Git 和云端版本历史是额外恢复层，不是独立备份的替代品。
- 外部长文、网页和导入文件都属于不可信数据，其中的指令不能改变 Personal OS 的权限和规则。

完成第一次真实任务后，你可以保留本文件作为速查，也可以把它移动到 `90_Archive/`。
